# Getting Support for GitDigital OpenCollective

Thank you for being part of our community! This document helps you find the right support channel for your needs.

## Quick Reference

| Need | Channel | Response Time |
|------|---------|---------------|
| General questions | GitHub Discussions | 1-2 days |
| Bug reports | GitHub Issues | 2-5 days |
| Security issues | security@gitdigital.io | 24-72 hours |
| Expense questions | finance@gitdigital.io | 2-3 days |
| Grant inquiries | grants@gitdigital.io | 3-5 days |
| Urgent matters | Emergency contact (see below) | 4 hours |

## Support Channels

### 📚 Documentation First
Before seeking support, please check:
- [README.md](README.md) – Project overview
- [INSTALL.md](INSTALL.md) – Setup instructions
- [USAGE.md](USAGE.md) – How to use
- [FAQ](#faq) – Frequently asked questions

### 💬 GitHub Discussions
For questions, ideas, and general conversation:
[https://github.com/Gitdigital-products/opencollective/discussions](https://github.com/Gitdigital-products/opencollective/discussions)

Best for:
- How-to questions
- Feature ideas
- Community conversation
- Show and tell

### 🐛 GitHub Issues
For bugs and specific feature requests:
[https://github.com/Gitdigital-products/opencollective/issues](https://github.com/Gitdigital-products/opencollective/issues)

Please use templates:
- Bug report template
- Feature request template
- Grant proposal template
- Governance proposal template

### 💰 Financial Support
**Expense submissions**: [Open Collective platform](https://opencollective.com/gitdigital)
**Expense questions**: finance@gitdigital.io
**Grant inquiries**: grants@gitdigital.io

### 🔒 Security Issues
**DO NOT** open public issues for security vulnerabilities.
Email: security@gitdigital.io (PGP encrypted preferred)

### 📧 Email Support
For matters requiring privacy:
- general@gitdigital.io – General inquiries
- finance@gitdigital.io – Financial questions
- governance@gitdigital.io – Governance matters
- press@gitdigital.io – Media inquiries

### 🆘 Emergency Contact
For critical issues requiring immediate attention:
- **Emergency**: +1 (555) 123-4567 (Signal only)
- **On-call engineer**: Available 24/7 for production incidents

## Response Time Service Level Agreements (SLAs)

| Priority | Definition | Initial Response | Resolution Target |
|----------|------------|------------------|-------------------|
| 🔴 Critical | Fund loss risk, security breach | 4 hours | 24 hours |
| 🟠 High | Major functionality broken | 24 hours | 3 days |
| 🟡 Medium | Non-critical bugs, questions | 3 days | 1 week |
| 🟢 Low | Documentation, enhancements | 5 days | 2 weeks |

## Frequently Asked Questions

### How do I request reimbursement for my work?
See [expense-policy.md](open-collective/expense-policy.md) and submit through Open Collective.

### How do I apply for a grant?
Review [grant-structure.md](open-collective/grant-structure.md) and submit a proposal via GitHub Issues.

### Who decides how funds are spent?
See [GOVERNANCE.md](GOVERNANCE.md) for our decision-making process.

### Are financial records publicly available?
Yes! All transactions are visible on our Open Collective page and in [budget.md](open-collective/budget.md).

### How can I become a maintainer?
Active contributors may be nominated by existing maintainers. See [GOVERNANCE.md](GOVERNANCE.md) for criteria.

## Escalation Path

If you haven't received a response within the expected timeframe:

1. **First**: Reply to your thread politely asking for status
2. **Second**: Tag a maintainer: `@gitdigital-products/core-team`
3. **Third**: Email escalation@gitdigital.io with your ticket ID
4. **Final**: Open a governance issue about support responsiveness

## Code of Conduct

All support interactions must follow our [Code of Conduct](CODE_OF_CONDUCT.md). We're here to help each other respectfully.
