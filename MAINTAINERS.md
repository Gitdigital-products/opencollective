## Maintainer Responsibilities

### All Maintainers
- Review issues and pull requests in a timely manner
- Participate in governance discussions
- Uphold the Code of Conduct
- Document decisions and rationale
- Mentor new contributors

### Core Maintainers (additional)
- Set project direction and priorities
- Approve budgets and fund releases
- Vote on governance changes
- Appoint new maintainers
- Represent the project publicly

### Technical Advisory Board (additional)
- Evaluate technical merit of grant proposals
- Review security implications of changes
- Advise on technical roadmap
- Participate in architecture decisions

### Finance Committee (additional)
- Review all expense submissions
- Prepare monthly financial reports
- Ensure policy compliance
- Monitor budget vs actuals

## Maintainer Selection

### Nomination Process
1. Existing maintainer nominates a contributor
2. Nominee must have 6+ months of active contribution
3. Community comment period (7 days)
4. Core Maintainers vote (unanimous required)

### Criteria
- Demonstrated expertise in focus area
- Consistent, quality contributions
- Alignment with project values
- Ability to work collaboratively
- Time availability for responsibilities

## Maintainer Removal

A maintainer may be removed for:
- Inactivity for 6+ months
- Code of Conduct violations
- Breach of trust or fiduciary duty
- Actions harming the project

Process:
1. Discussion among Core Maintainers
2. 14-day community comment period
3. Core Maintainers vote (2/3 majority)

## Meeting Cadence

- **Core Maintainers**: Weekly (30 min)
- **Technical Advisory Board**: Bi-weekly (60 min)
- **Finance Committee**: Monthly (45 min)
- **All Maintainers**: Quarterly (90 min)

Meeting notes are public in the `/meetings` directory.
