🚀 Getting Started

Follow these steps to integrate the zkSolana KYC Compliance SDK into your project.

📋 Prerequisites: Rust & Solana CLI, Anchor Framework, Node.js (v18+) and Yarn.

Installation

# Clone the repository
git clone https://github.com/Gitdigital-products/zk-solana-kyc-compliance-sdk.git
cd zk-solana-kyc-compliance-sdk

# Install dependencies
yarn install

# Build the on-chain program
anchor build

Initialize a Compliant Mint

1
Set up the SDK in your TypeScript project 
import { ComplianceSDK } from '@gitdigital/solana-kyc-sdk';
import { Connection, Keypair } from '@solana/web3.js';

const connection = new Connection('https://api.mainnet-beta.solana.com');
const wallet = Keypair.generate(); // In production, use a secure wallet

const sdk = new ComplianceSDK(connection, wallet);
2
Create a token mint with Transfer Hook extension 
const mint = await sdk.createCompliantMint({
  name: "My Regulated Token",
  symbol: "RWA",
  decimals: 6,
  authority: wallet.publicKey, // Mint authority
  transferHookProgramId: sdk.programId, // Your deployed hook program
});

Validating Transfers

Once the mint is created, every transfer is automatically checked by the Transfer Hook. The hook queries the on‑chain Compliance Registry to verify that both sender and receiver have the required KYC flags. If not, the transaction fails at runtime.

// This transfer will succeed only if both parties are compliant
await transferTokens(mint, fromAccount, toAccount, amount);
