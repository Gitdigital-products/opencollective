🏗️ Architecture

The SDK is composed of three primary layers that work together to enforce compliance at the token level.

1. On‑Chain Program (Rust/Anchor)

This program contains the core logic for the Transfer Hook and validation of identity metadata. It is deployed as a Solana program (using Token-2022 extensions) and is invoked automatically whenever a token transfer occurs on a compliant mint.

Key responsibilities:

  • Intercepting transfers via the Transfer Hook interface.
  • Querying the Compliance Registry to check sender/receiver KYC status.
  • Enforcing rules (e.g., allow only verified accounts, apply transfer limits).

2. Compliance Registry

A public, versioned on‑chain directory of trusted KYC/AML providers and their attestations. The registry maps wallet addresses to compliance flags (e.g., "level-1 KYC", "accredited investor", "sanctions check passed"). Providers update these flags after off‑chain verification.

Key responsibilities:

  • Storing provider public keys and their rule sets.
  • Maintaining a history of attestations for auditability.
  • Allowing token programs to read compliance data during transfers.

3. TypeScript SDK

A high‑level library for frontend and backend integration. It abstracts the complexity of interacting with the on‑chain program and registry, providing simple methods to create compliant mints, register users, and query compliance status.

Key responsibilities:

  • Initializing compliant mints with the correct extensions.
  • Submitting identity attestations to the registry.
  • Monitoring transfer compliance and debugging failures.

📊 Compliance Flow Diagram

[User] → (1) KYC with Provider → (2) Provider signs attestation → (3) Attestation stored in Registry
↓
[Transfer initiated] → Transfer Hook triggered → Query Registry for sender & receiver → Approved/Denied

Token Extensions Used

  • Transfer Hook: Core extension that invokes our program on every transfer.
  • Permanent Delegate: Enables administrative controls like freezing or recovering assets in compliance with regulations.
  • Metadata Pointer: Attaches off‑chain metadata (name, symbol) to the mint.
